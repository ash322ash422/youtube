  1. Open Tableau Desktop

    Launch Tableau. On the Start page, under "Connect", click Microsoft Excel (Superstore is an .xls file).

  2. Navigate to the File

    Browse to Documents/My Tableau Repository/Datasources/ and open Sample - Superstore.xls.

  3. Select the Orders Sheet

    In the Data Source tab, you'll see three sheets: Orders, People, Returns. Drag Orders to the canvas. This is our primary table — 9,994 rows.

  4. Verify the Data

    Scroll the preview at the bottom. Check that Sales and Profit are green (Measures) and Category, Region are blue (Dimensions). If not, right-click and change data role.

  5. Create New Sheet

    Click the orange Sheet 1 tab at the bottom. You're now in the worksheet view — the Tableau workspace where all chart building happens.

💡

Pro Tip: Create a Data Extract

Before building, go to Data → [Superstore] → Extract Data. Save the .hyper file. This will make every interaction 3–5× faster since Tableau reads from its own optimised engine rather than the Excel file.

How to Build KPI Cards in Tableau

  1. Double-click "Sales" in the Measures pane

    Tableau drops SUM(Sales) into the view as a single number. This is your starting point.

  2. Set Mark Type to "Text"

    In the Marks card, click the dropdown and choose Text. The number now fills the canvas cleanly.

  3. Format the Number

    Right-click SUM(Sales) on the Text shelf → Format → Numbers → Currency (Custom). Set to 2 decimal places, prefix $, and enable suffix "K" or "M" for scale.

  4. Add Year-over-Year Comparison

    Drag Order Date to the Filters shelf. Create a calculated field: YEAR([Order Date]) = YEAR(TODAY())-1. Add a reference line or use a table calculation for % change.

  5. Repeat for Profit, Orders (COUNTD), and Profit Margin

    Create 3 more sheets. For Orders use COUNTD([Order ID]). For Profit Margin use the calculated field SUM([Profit]) / SUM([Sales]), formatted as %.

Tableau Calculated Fields for KPIs
// Profit Margin % SUM([Profit]) / SUM([Sales]) // Format as: Percentage, 1 decimal place // Year-over-Year Growth (SUM([Sales]) - LOOKUP(SUM([Sales]), -1)) / ABS(LOOKUP(SUM([Sales]), -1)) // Use as a Table Calculation, compute along: Year // Total Orders (distinct) COUNTD([Order ID]) // Average Order Value SUM([Sales]) / COUNTD([Order ID])
Step 04 — Build the Charts

Monthly Sales Trend — Line Chart

This is the spine of your dashboard. A monthly trend chart answers "are we growing?" and instantly reveals seasonality, dips, and peaks. We'll show 4 years overlaid to spot patterns.

Monthly Sales Trend 2021–2024

Columns: MONTH(Order Date) | Rows: SUM(Sales) | Color: YEAR(Order Date)
Line Chart

  1. Create a New Worksheet (Ctrl+M)

    Name it "Sales Trend" by double-clicking the tab. Always name your sheets — they appear as titles in the dashboard.

  2. Drag "Order Date" to Columns

    Tableau auto-creates a YEAR hierarchy. Click the + on YEAR(Order Date) to expand to QUARTER, then again to MONTH. Right-click and choose Month (continuous) for a smooth line.

  3. Drag "Sales" to Rows

    SUM(Sales) appears. Tableau draws a single aggregated line. Perfect starting point.

  4. Break by Year for Color

    Drag Order Date to the Color shelf in the Marks card. In the dropdown, select YEAR (discrete). You now get 4 colored lines, one per year.

  5. Add a Trend Line

    Right-click the chart → Trend Lines → Show Trend Lines. Choose Linear. This reveals the growth direction across years.

  6. Format: Remove Gridlines, Keep Reference Lines

    Format → Lines → set Grid Lines to None. Right-click the Sales axis → Add Reference Line → Average. This shows the mean monthly sales as a benchmark dashed line.

📌

Key Insight from This Chart

You'll notice a consistent Q4 spike every year (October–November). This is the holiday season effect. Your dashboard consumer needs to know this context — add an annotation with Ctrl+click on the spike point.

Step 05 — Build the Charts

Sales by Region — Bar Chart

A horizontal bar chart comparing sales across the 4 US regions. We'll sort descending so the best-performing region is always on top — a small detail that makes a big readability difference.

Sales & Profit by Region

Rows: Region | Columns: SUM(Sales), SUM(Profit) | Color: Segment
Bar Chart

  1. Drag "Region" to Rows

    You get 4 rows: Central, East, South, West.

  2. Drag "Sales" to Columns

    Tableau draws horizontal bars. Click the Sort Descending button (↓) in the toolbar to rank by highest sales.

  3. Add Profit as a second axis

    Drag Profit to the right side of the existing bar chart until you see a green dashed line — drop it to create a dual axis. Right-click the second axis and choose "Synchronize Axis".

  4. Color by Segment

    Drag Segment to the Color shelf. Choose a diverging palette from the color editor — use blue for Consumer, orange for Corporate, teal for Home Office.

  5. Add Data Labels

    Click the Label shelf in Marks → check "Show mark labels" → set to Min/Max only to avoid clutter.

Step 07 — Build the Charts

Discount vs Profit Scatter Plot

This is the most analytical chart in the dashboard — and the most revealing. It answers: "Is our discounting strategy hurting profitability?" The answer, in the Superstore data, is a clear and painful yes.

Discount Rate vs Profit by Sub-Category

X: AVG(Discount) | Y: SUM(Profit) | Size: SUM(Sales) | Color: Category
Scatter Plot

  1. Drag "Discount" to Columns, "Profit" to Rows

    Change the aggregation of Discount to AVG (right-click → Measure → Average). Keep Profit as SUM.

  2. Add Sub-Category to Detail

    Drag Sub-Category to the Detail shelf in Marks. This disaggregates the data — now each dot is a sub-category, not a single aggregate.

  3. Size = Sales, Color = Category

    Drag Sales to Size (bigger dot = more revenue). Drag Category to Color. You can now see Furniture (orange), Technology (blue), Office Supplies (green) scattered across the chart.

  4. Add a Reference Line at Discount = 0.2

    Right-click the X-axis → Add Reference Line → Constant = 0.2. Discounts above 20% are where the losses cluster. This line makes the threshold visual and unmistakable.

  5. Add Trend Line

    Right-click inside the chart → Trend Lines → Show Trend Lines. The negative slope confirms: more discount = less profit. A powerful story told by a single line.

🔍 Key Insight: The Discount Trap

  • Sub-categories with 0% discount average +22% profit margin
  • Sub-categories with >40% discount average -38% profit margin
  • Tables and Bookcases are consistently discounted at 30–45% — driving the entire Furniture category into low margins
  • Technology items (Phones, Accessories) maintain high margins even with moderate discounts
  • Recommendation: Cap discounts at 20% across Furniture sub-categories immediately

Step 09 — Assemble

Building the Dashboard Canvas

Now that all 6 chart types are built as individual worksheets, it's time to assemble them into a single, cohesive dashboard. This is where layout thinking becomes critical.

📐

Dashboard Size Setting

Go to Dashboard → Size → Fixed Size → 1200 × 900 pixels. This is optimised for laptop screens (standard 1366×768 with browser chrome). For TV/large monitor presentations, use 1920×1080.

Layout Blueprint

ROW 1: [ KPI: Total Sales ] [ KPI: Total Profit ] [ KPI: Orders ] [ KPI: Margin ] ← 4 columns, ~120px tall
ROW 2: [ Sales Trend Line — SPANS 2/3 width ] | [ Category Donut — 1/3 width ]
ROW 3: [ Region Bar Chart — 1/3 ] | [ Sub-Category Bar — 1/3 ] | [ Discount Scatter — 1/3 ]
ROW 4: [ State Map — FULL WIDTH ] ← ~280px tall

  1. Click "New Dashboard" tab (+ icon at bottom)

    Set fixed size to 1200×900. In the left panel, you'll see all your named worksheet tabs listed.

  2. Drag "Sales Trend" to the canvas first

    It fills the whole canvas. Don't worry — drag the others and they'll tile automatically. The grey guidelines show where Tableau will snap each sheet.

  3. Use Tiled Layout for main structure

    Ensure "Tiled" is selected (not Floating) in the Layout panel. Drag the KPI sheets to the top row. Tableau creates equal columns automatically.

  4. Resize Rows and Columns

    Hover over the edge of any tile — a grey resize handle appears. Drag to adjust proportions. The trend chart should be the largest single element (Rule: largest = most important).

  5. Add Title and Logo (Floating)

    Add a Text Object (from the bottom left panel) as a Floating element over the top of the dashboard. Type your dashboard title. Set font to bold 22px. Add a horizontal rule below it using a blank floating sheet with a bottom border.

Step 10 — Assemble

Actions, Filters & Interactivity

A Tableau dashboard without actions is just a static poster. Actions are what transform it into an interactive analytics experience. Here's how to add every key interaction.

Filter Action (Click to Filter)

  1. Dashboard → Actions → Add Action → Filter

    Source: "Sales Trend" sheet. Target: ALL sheets. Run action on: Select. This means clicking a month bar filters every chart to that month's data.

  2. Set "Clearing the selection will" to: Show all values

    This resets the dashboard when the user clicks an empty area — critical for usability.

Highlight Action (Hover to Highlight)

  1. Dashboard → Actions → Add Action → Highlight

    Source: Region Bar. Target: Map. Target highlighting: Selected fields → Region. Hovering over "West" bar now glows West states on the map.

  2. URL Action for Drill-Down

    Add a URL action on the Category sheet. URL: https://www.tableau.com/search?q=<Category> — opens a new tab with more context when users click a category.

Quick Filters (Dashboard-Level)

  1. Add a Year Filter

    Show the "Order Date" filter from any sheet. Right-click it → Apply to Worksheets → All Using This Data Source. Format as a radio button list showing 2021, 2022, 2023, 2024.

  2. Add a Segment Filter

    Show the "Segment" filter. Format as Checkbox (multi-select). Place it in the top-right of the dashboard in a floating container.

  3. Add a Region Filter

    Show the "Region" filter. Format as a dropdown. Stack filters vertically on the right side — consistent positioning trains users to look there for controls.

✨

The Interactivity Test

Click every element on your dashboard and ask: "Did something useful happen?" If clicking the Central region bar should filter the map to Central states, and it doesn't — add an action. Every click should reveal insight.
